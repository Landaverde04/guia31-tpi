Conclusiones

¿A partir del ejemplo 2 y ejercicios desarrollados hasta el momento que puede concluir?

Con este ejercicio entendí mejor cómo funciona la estructura de un archivo XML. Al inicio solo tenía un país con un departamento, pero al ir agregando más información (como monedas, área territorial, idiomas y más departamentos) me di cuenta de que XML se organiza como un árbol, donde cada cosa debe estar en el nivel correcto para que tenga sentido.

También noté que los atributos sirven bien para detalles cortos (como el nombre o el id de un departamento), mientras que las etiquetas son mejores para datos que tienen contenido más amplio, como los nombres, la población o la altura.

Conclusiones CSS

Con este ejercicio me di cuenta de cómo un archivo XML puede tomar forma y ser mucho más fácil de leer cuando se le aplica CSS. Al principio el XML solo se veía como texto plano, pero al darle estilos pude resaltar cada parte (país, monedas, idiomas, departamentos) de manera distinta. También aprendí que se pueden usar diferentes formas de mostrar la información: por ejemplo, poner algunos datos en línea, otros en bloques, cambiar colores, bordes o fondos para distinguir secciones.
